Java-进阶-JavaWeb

1. 基本概念

1.1 前言

对于web开发而言：

  • web就是网页的意思：www.baidu.com
  • 静态web
    • html,css
    • 提供给所有人看的网页数据始终不会发生变化
  • 动态web
    • 淘宝，几乎是所有的网站
    • 提供给每个人看的数据在不同时间，不同地点都不一样
    • 技术栈：Servlet/JSP,ASP,PHP

1.2 web 应用程序

web应用程序：可以提供浏览器访问的程序；

  • a.html、b.html......多个web资源，这些web资源可以被外界访问，对外界提供服务；

    • 能访问到的任何一个页面或者资源，都存在于这个世界的某一个角落的计算机上。
    • URL 统计资源定位符来找到它们

  • 这个统一的web资源会被放在同一个文件夹下，web应用程序–>Tomcat：服务器

  • 一个web应用由多部分组成（静态web,动态web）

    • html,css,js
    • jsp,servlet
    • java程序
    • jar包
    • 配置文件（properties）

  • web应用程序编写完毕后，若想提供给外界访问：需要一个服务器来统一管理；

    • 常用的就是Tomcat 服务器

1.3 静态web

  • *.htm *.html这些都是网页的后缀，如果在服务器上一直存在这些东西，我们就可以直接进行读取

  • 静态web存在的缺点
    • Web页面无法动态更新，所有用户看到的都是同一个页面（html 静态资源）
      • 轮播图 ，点击特效：伪动态
      • JavaScript【实际开发中用得最多：伪动态】
      • VBScript ： 伪动态
      • 存放的是静态资源
    • 它无法和数据库交互

1.4 动态web

  • 作用：页面会动态展示： “Web的页面展示的效果因人而异”；

缺点：

  • 加入服务器的动态web资源出现了错误，我们需要重新编写我们的后台程序,重新发布；
    • 停机维护

优点：

  • Web页面可以动态更新，所有用户看到都不是同一个页面
  • 它可以与数据库交互 （数据持久化：注册，商品信息，用户信息……..）

4. HTTP协议

4.1 什么是HTTP

HTTP（超文本传输协议）是一个简单的请求-响应协议，它通常运行在TCP之上。

  • 文本：html,字符串….
  • 超文本：图片，音乐，视频，定位，地图
  • 端口号：80
  • 应用层协议

Https：安全的HTTP

  • 端口号：443
  • 应用层协议

4.2 http 的版本

  • http1.0

    • HTTP/1.0：客户端可以与web服务器连接后，只能获得一个web资源，断开连接（短链接）

  • http2.0

    • HTTP/1.1：客户端可以与web服务器连接后，可以获得多个web资源（长链接）

  • 具体可以参考 ： HTTP的详细博客

7.4 Session(重点)

什么是Session？

  • 服务器会给每一个用户（浏览器）创建一个Session对象
  • 一个Session独占一个浏览器，只要浏览器没有关闭，这个Session就存在
  • 用户登录之后，整个网站它都可以访问–》保存信息，保存购物车信息

Session和Cookie的区别

  • Cookie是把用户的数据写给用户的浏览器，浏览器保存（可以保存多个）
  • Session把用户的数据写到用户独占Session中，服务器端保存（保存重要的信息，减少服务器资源的浪费）
  • Session对象有服务器端创建

使用场景

  • 保存一个用户登录的信息
  • 淘宝购物车的信息
  • 在整个网站中经常会使用到的数据，我们将它保存在Session中
