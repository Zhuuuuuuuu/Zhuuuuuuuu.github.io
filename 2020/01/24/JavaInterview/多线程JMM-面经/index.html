1. 多核并发缓存架构

  • 多任务处理在现代计算机操作系统几乎已经是一项必备的功能。
  • 处理器上的寄存器的读写的速度比内存快几个数量级，为了解决这种速度矛盾，在它们之间加入了高速缓存。
  • 高速缓存在多线程下带来了一些问题
    • 缓存一致性问题：当多个处理器的运算任务都涉及同一块主内存区域的时候，将可能导致各自的缓存不一致的情况，如果发生了这种情况，需要各个处理器遵循一些协议，在读写时要根据协议来操作（MSI,MESI）
      • 早期采用总线加锁
      • 现在采用（MSI,MESI）缓存一致性
    • 指令重排序问题：为了使得处理器内部的运算单元能被尽量充分利用，处理器可能会对输入的代码及逆行乱序执行（Out-Of-Order Execution）优化，处理器会在计算结果之后对乱序执行后的结果及逆行重组，保证这个乱序结果和顺序结果的执行结果一致。

2. JMM

  • Java的线程内存模型
  • JVM内存模型指的是JVM的内存分区；而Java内存模式是一种虚拟机规范。

2.1 主内存和工作内存

  • 所有的变量都存储在主内存之中，每个线程还有自己的工作内存
  • 工作内存存储在高速缓存或者寄存器之中，保存了该线程使用的变量的主内存副本拷贝。

关于JMM的一些同步约定：

  • 线程解锁前，必须把共享变量都刷回主内存
  • 线程加锁前，必须读取主内存中最新的值
  • 加锁和解锁是同一把锁

2.2 内存交互操作

  • Java 内存模型定义了 8 个操作来完成主内存和工作内存的交互操作。

  1. read: 从主内存中读取数据
  2. load: 将read之后的值放到工作内存中
  3. use : 使用工作内存中的值
  4. assign: 将计算好的值重新赋值到工作内存中
  5. store: 将修改后的值放回到主内存
  6. write : 将放回到主内存的值写入到主内存中去
  7. lock : 将主内存变量加锁，标识位线程独占的状态
  8. unlock : 对主内存变量解锁

2.3 volatile 可见性底层实现原理

  • 关键在于lock前缀指令（汇编语言），它的作用是将本处理器的缓存立马同步回主内存，该写入动作也会引起其他处理器或者别的内核的无效化（Invalidate）其缓存。

    • 这种操作就相当于对缓存中的变量做了一次起那面介绍了Java内存模型的“store” “write” 操作。
    • 同时开启了MESI缓存一致性协议

  • lock 前缀指令相当于插入了一个内存屏障（Memory Barrier）

  • 在 store 操作之前会加一把锁(lock操作)，在write 之后才会unlock

2.4 并发编程的三大特性

并发编程的三大特性

  • 可见性
  • 原子性
  • 有序性

Volatile是Java虚拟机提供的轻量级的同步机制

  • 保证可见性
  • 不保证原子性
  • 禁止指令重排(有序性)
